# Contributing to K413MP3R4's Projects

Merci de votre intérêt pour contribuer à mes projets ! 🙏

## Comment Contribuer

### Signaler des Bugs
- Ouvrez une **GitHub Issue** avec un titre clair
- Décrivez le comportement attendu vs observé
- Incluez des étapes de reproduction si possible

### Proposer des Améliorations
- Ouvrez une **GitHub Discussion** ou **Issue** avec le tag `enhancement`
- Expliquez le bénéfice et le cas d'usage

### Soumettre du Code
1. **Fork** le dépôt
2. Créez une branche pour votre changement : `git checkout -b fix/my-issue`
3. Commitez avec le **DCO sign-off** : `git commit -s -m "Fix: description"`
4. Poussez vers votre fork et ouvrez une **Pull Request**

```
(a) The contribution was created in whole or in part by me and I

(b) The contribution is based upon previous work that, to the best

    work with modifications, whether created in whole or in part

    permitted to submit under a different license), as indicated
    in the file; or

(c) The contribution was provided directly to me by some other
    person who certified (a), (b) or (c) and I have not modified

(d) I understand and agree that this project and the contribution
    are public and that a record of the contribution (including all
    personal information I submit with it, including my sign-off) is
    maintained indefinitely and may be redistributed consistent with
    this project or the open source license(s) involved.
```

### Signer vos Commits

Utilisez `-s` ou `--signoff` lors du commit :

```bash
git commit -s -m "Add: new feature"
```

Cela ajoute automatiquement votre ligne de signature :

```
Signed-off-by: Your Name <your.email@example.com>
```

## Directives de Code

- Respectez le style existant du projet
- Écrivez du code lisible et bien documenté
- Testez vos changements avant de soumettre
- Évitez les changements non liés au scope de la PR

## Licence
